[b]Title:[/b] [Tool] Factorio AI Tools - Claude Desktop MCP Server (Blueprint Decoding, Clusterio, AST Search)

Hey engineers,

I've been working on a massive open-source AI integration for Factorio modding and server management, and I wanted to share it with the community. 

[b]Factorio AI Tools[/b] is a Model Context Protocol (MCP) server that plugs directly into Claude Desktop. It acts as a massive Retrieval-Augmented Generation (RAG) pipeline for your AI assistant, giving it deep, searchable access to the actual mechanics of the game. 

While stopping AI hallucinations is great, that's only a fraction of the benefit. By embedding the codebase, wiki, and forums natively, this tool:
[list]
[*] [b]Saves Your Time:[/b] You no longer have to manually copy-paste massive API pages, plugin architectures, or forum threads into the chat to give the AI context.
[*] [b]Understands Architecture:[/b] The AI gets a "bird's eye view" of the entire Factorio modding stack (Data vs Control phases) and complex multi-server setups (Clusterio IPC messaging).
[*] [b]Discovers Undocumented Fixes:[/b] It can recall obscure community workarounds and edge-cases directly from the Modding Help forums that standard AI training data completely misses.
[*] [b]Version Accuracy:[/b] It natively understands the difference between Factorio 1.1 and 2.0 APIs, preventing it from suggesting deprecated functions.
[/list]

We wanted to build something that wasn't just "scraped slop", but an actual high-performance engineering tool.

[size=150][b]Why We Built It This Way (RAG vs. Grep vs. Web Search)[/b][/size]
Many AI coding agents rely solely on live web searches or local `grep`. We chose a hybrid, pre-embedded RAG approach for very specific reasons:
[list]
[*] [b]Conceptual Search over Exact Precision:[/b] Tools like `grep` are surgical, deterministic, and the industry standard for exact file navigation. However, they fail when you don't know the exact keyword. Our RAG pipeline maps the [i]meaning[/i] of your query, allowing the AI to find conceptually related code (e.g., "handle user authentication" or "fluid box connection") even if the exact variable names differ.
[*] [b]Zero-Configuration Fast Lookups:[/b] Live web searches for docs are always up-to-date and give you community support, but they are slow and require you to chunk and embed massive volumes of external documentation on the fly for every single query. We use LanceDB because it is an embedded, zero-configuration format built directly on the file system. It enables rapid similarity searches without any pipeline overhead or heavy external database clusters.
[/list]

[size=150][b]Technical Highlights[/b][/size]
[list]
[*] [b]Embedded LanceDB Vector Database:[/b] Ultra-fast semantic search running locally. No bloated Docker containers or external cloud databases required.
[*] [b]Semantic Code Chunking:[/b] It doesn't just read raw text. It parses codebases into abstract syntax trees (AST) to distinguish between class declarations, method definitions, and configuration files, meaning the AI actually understands the architecture of the code it reads.
[*] [b]Native Blueprint Decoding:[/b] Built-in Zlib decompression and Base64 validation to securely parse Factorio's [c]0eNq...[/c] blueprint strings directly into formatted JSON arrays for the AI to manipulate.
[*] [b]BGE Dense Vectors:[/b] Uses the top-tier [c]BAAI/bge-base-en-v1.5[/c] embedding model with automatic GPU hardware acceleration.
[/list]

[size=150][b]What it does today[/b][/size]
